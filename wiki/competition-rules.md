# Competition Rules

本文件记录 WRC / 机器人比赛“园区管理岗”场景的规则背景和视觉任务含义。视觉系统架构设计详见 [[architecture]]，类别清单详见 [[class-list]]。

## 比赛场景概述

本项目来自 WRC / 机器人比赛中的“园区管理岗”场景。机器人需要在园区环境中完成与视觉相关的任务，并根据识别结果进行语音提醒、靠近目标、拾取垃圾或分类投放。

视觉相关任务包括：

1. 禁带品检查。
2. 不文明行为提醒。
3. 垃圾识别与拾取分类。

## 任务一：禁带品检查

### 场景

入园人员推行露营车通过安检区。露营车会在安检区短暂停留，视觉系统需要在有限时间内识别禁带品。

目标是准确识别禁带品，并触发对应语音提示。

### 禁带品类别

- 喷雾罐。
- 滑板。
- 儿童滑板车。
- 烧烤炉 / 炭火炉。
- 音响。
- 卡式炉。
- 轮滑鞋。
- 扩音喇叭。

### CV 含义

- 这是以 object detection 为主的任务。
- 需要在短时间窗口内完成识别。
- 要重点控制误报，不能把普通物品误判成禁带品。
- 适合小模型快速识别 + 低置信度大模型复核。

## 任务二：不文明行为提醒

### 目标

识别踩踏草坪、吸烟、占用消防通道、站立 / 躺在长椅上等行为。机器人需要靠近不文明行为人员，并进行对应语音提醒。

### 行为类别

- 踩踏草坪。
- 吸烟。
- 占用消防通道。
- 站立 / 躺在长椅上。

### CV 含义

- 这不是单纯 object detection。
- 需要结合 person、bench、草坪 / 消防通道区域、姿态、动作、上下文关系。
- 可能需要规则判断、VLM 复核、pose estimation、tracking 或多帧时序信息。
- 不应在第一版中简单假设 YOLO 能直接识别完整行为。

## 任务三：垃圾识别与拾取分类

### 目标

识别园区内散落垃圾，机器人需要靠近并拾取垃圾，然后投入正确垃圾桶。

### 垃圾类别

- 硬挺外卖袋。
- 一次性食品盒。
- 塑料饮料瓶。
- 食品塑料包装袋。
- 纸团。
- 空烟盒。

### CV 含义

- 第一阶段可先做 object detection，返回 bbox。
- 如果后续抓取需要更精确区域，可能需要 segmentation。
- 需要记录垃圾类别和位置，供机器人靠近、抓取和分类。

## 计分和风险

比赛对视觉系统的关键影响：

- 误识别、误提示会导致扣分。
- 禁带品不能把普通物品误判成禁带品。
- 不文明行为不能对正常行为或错误对象提醒。
- 系统不能只依赖单帧低置信度结果直接触发动作。
- 架构中必须考虑 confidence 阈值、多帧确认、疑似结果、低置信度复核、日志回流。

## 对架构的影响

三类任务的逻辑不同：

- 禁带品：短时间窗口，小模型优先，多帧确认，低置信度复核。
- 垃圾：可静止识别，先 detection，后续视抓取需求考虑 segmentation。
- 不文明行为：语义和关系判断更复杂，不应简单当作普通物体检测。
