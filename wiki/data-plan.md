# Data Plan

本文件记录第一阶段数据采集、标注、负样本和 baseline 数据规模想法。当前内容是初步计划，不代表数据已经存在。

## 第一阶段目标

建立一个可用于 YOLO11n / YOLO11s baseline 试验的小规模数据集，用于验证：

- 类别是否可区分。
- 标注规范是否合理。
- 小模型是否能达到基本实时识别效果。
- 哪些类别需要更多数据或更复杂模型。

## 数据来源设想

待确认的数据来源可能包括：

- 自行拍摄的模拟园区场景。
- 比赛训练场或实景测试环境。
- 队内搭建的禁带品和垃圾摆拍样本。
- 合规来源的公开图片。
- 合成或增强数据。

所有数据来源需要确认版权、比赛规则和使用边界。

## 标注计划

第一阶段优先考虑 object detection 标注：

- 禁带品：标注物品框。
- 垃圾：标注垃圾物体框。
- 人员、长椅等辅助类别：根据任务需要标注。
- 不文明行为：先评估是否能通过目标框和规则表达，不能直接假设为普通检测类别。

待确认：

- 标注工具。
- 标注格式。
- 类别命名规范。
- train / val / test 划分方式。
- 多人标注一致性检查方式。

## 负样本

需要准备负样本，避免模型把普通场景误识别为目标类别。

负样本例子：

- 没有禁带品的入园人员。
- 普通背包、手提袋、露营物品。
- 干净地面和正常园区道路。
- 正常坐在长椅上的人员。
- 非草坪区域的人员活动。
- 类似但不违规的物品。

## baseline 数据规模想法

第一阶段可以先建立小规模验证集，具体规模待确认。一个可讨论的起点：

- 每个禁带品类别至少几十到数百张有效样本。
- 每个垃圾类别至少几十到数百张有效样本。
- 每类不文明行为先收集足够样本用于判断任务形式。
- 每个核心场景都需要负样本和困难样本。

该规模只是 planning 起点，不是最终训练标准。

## 数据目录建议

大型数据不进入 Git。建议后续在本地或外部存储中组织：

- `data/raw/`：原始图片、视频或帧。
- `data/images/`：整理后的训练图片。
- `data/labels/`：标注文件。
- `data/videos/`：原始或测试视频。
- `data/exports/`：标注工具导出包。

上述目录已在 `.gitignore` 中忽略。
